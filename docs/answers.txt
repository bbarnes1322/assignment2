4) Two reasons why branches are used
Branches let you work on new features/bug fixes without breaking the main branch.
Branches allow parallel work by multiple people and make merging/review easier.

5) Difference between git log and git status
git status shows the current state of your working directory and staging area (modified/staged/untracked files).
git log shows the commit history (past commits with messages, authors, and dates).

8) OOP Concepts
(I) Inheritance: A derived class inherits fields/methods from a base class and can extend or override behavior.
(II) Polymorphism: The same function call/interface can behave differently depending on the actual object type like virtual method overriding.
(III) Encapsulation: Hiding internal data and requiring interaction through class methods, private data + public methods.

9) What is the main difference between the “Dictator and Lieutenants” workflow and the “Integration manager” workflow?

The Integration Manager workflow allows developers to push their work to their own public repositories and then submit it to a central repository. An integration manager reviews the changes and merges them into the main project.

In the Dictator and Lieutenants workflow, only one main maintainer, the dictator, can push changes to the central repository. Lieutenants manage groups of developers and review their changes before passing them to the dictator. This spreads the responsibility across multiple people instead of everything going through one manager.

10) How would a team of 100 developers benefit from following the “Dictator and Lieutenants” workflow instead of the “Centralized” workflow?

With a team of 100 developers, a centralized workflow could cause many conflicts because everyone would be pushing changes directly to the same repository. This could slow down development and make it harder to manage changes.

The Dictator and Lieutenants workflow helps organize the team by having developers submit their work to lieutenants first. The lieutenants review and combine the changes before sending them to the main maintainer. This reduces conflicts and makes it easier for large teams to manage their work.
